MVP Limitations & Known Constraints

Transparency about what MailGen can and cannot do

What This MVP Delivers

  • AI-powered email generation from documents
  • Multi-format document analysis (.txt, .pdf, .docx)
  • End-to-end user flow (upload → generate → send)
  • Email scheduling for future delivery
  • Gmail SMTP integration with App Passwords
  • File attachment support (upload, URL, folder)

Known Limitations

Limitation Impact Why Acceptable for MVP
In-Memory Scheduling Scheduled emails are lost if the server restarts. No persistent storage. Proves scheduling concept; database integration planned for production
Gmail Only Only supports Gmail SMTP with App Passwords. No Outlook, Yahoo, etc. Gmail covers ~30% of global email; validates core sending workflow
No User Accounts No login, history, or personalization across sessions MVP tests whether core value (AI generation) works, not user management
No Email Preview/Edit Users cannot review or modify the AI-generated email before sending Validates AI quality assumption; preview is a Phase 2 feature
Single Recipient Can only send to one email address at a time Primary use case (job applications) is 1:1 communication
AI Quality Variance Output quality depends on document clarity and topic specificity. Occasional hallucination possible. Inherent LLM limitation; mitigated by document-grounding via Agent 1
No Offline Support Requires internet for AI (Gemini API) and email (Gmail SMTP) Cloud APIs are the standard approach; offline not needed for validation
Processing Time Email generation takes 15–40 seconds due to multi-agent processing Still 10× faster than manual workflow; optimization possible later

Deliberately Not Built

These features were considered but intentionally excluded to keep the MVP focused on core value validation:

Email Templates Library
Would add complexity without testing core AI generation hypothesis
Multi-Language Support
English-only validates the core workflow first
Analytics Dashboard
Metrics tracking is a post-PMF feature
OAuth Gmail Login
App Passwords are simpler to implement and test
Mobile App
Web interface is responsive; native app is premature
Batch Email Sending
1:1 personalized sending is the core differentiator

How We Handle Failures

Invalid Document

Unsupported formats, corrupted files, or empty documents are caught with clear error messages before AI processing begins.

AI Generation Failure

If the AI produces empty/short output or API fails, the user gets a specific error message instead of a broken email being sent.

Send Failure

Invalid credentials, wrong email format, or SMTP errors are caught and reported. Scheduled emails track "failed" status with error details.
